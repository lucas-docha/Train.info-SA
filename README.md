Título do projeto: Train.info
Objetivo do projeto: Criar um sistema para controle do ferrorama
Contexto: O projeto foi idealizado na sala de aula do curso técnico da turma DS24 — M7/T9, com o objetivo de aprimorar os conhecimentos sobre back-end, front-end, banco de dados e IoT.
Funcionalidades principais: Crud's trem, rota, manutenção, sensores, notificações e usuários. O projeto conta com tela de login, cadastro, dashboard, criação de relatórios.
Tecnologias utilizadas: VS Code, Github, Arduino IDE. Linguagens utilizadas: PHP, MySQL, HTML, MQTT.
Equipe de desenvolvimento: Veloso Productions - Lucas Docha, Caio Ferreira, Henrique Augusto e Matheus Gabriel
Estrutura do repositório: 
Train.info-SA/
├─ css/                    → Arquivos de estilo (visual do site)
├─ experimental/           → Testes e funcionalidades experimentais
├─ manutencoes/            → CRUD de manutenção de trens
├─ notificacoes/           → Sistema de notificações
├─ relatorios/             → Geração de relatórios
├─ rotas/                  → CRUD de rotas e itinerários
├─ sensores/               → CRUD e leitura de sensores
├─ trens/                  → CRUD de trens
├─ usuarios/               → Login, cadastro e perfil de usuários
├─ banco_SA.sql            → Script para criar o banco de dados
├─ config.php              → Configurações gerais
├─ dashboard.php           → Tela principal (dashboard)
├─ index.php               → Página inicial
├─ login.php               → Página de login
├─ logout.php              → Página de sair da sessão
└─ sobre.php               → Página “Sobre o sistema”
